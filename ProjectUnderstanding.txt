CQRS = Command Query Responsibility Segregation

Meaning:

Commands = change data (Create, Update, Delete)

Queries = read data (Get something)

We keep them separate because:

Command logic is complex (validation, domain rules, saving)

Query logic is optimized for fast reading

📌 So CQRS segregates read and write responsibilities.


MediatR helps us send commands or queries to the correct handler without knowing who handles it.

RabbitMQ is low-level.

MassTransit is a .NET abstraction over RabbitMQ.

MassTransit gives you:

✔ Automatic queue creation
✔ Retries
✔ Dead-letter queues
✔ Serialization
✔ Dependency injection
✔ Consumer lifecycle

Think of it as:

“ASP.NET Core for messaging”

COMPLETE FLOW (READ SLOWLY)
1. User clicks "Checkout"
2. Basket Service publishes BasketCheckoutEventV2
3. RabbitMQ stores message
4. Ordering Service consumer receives event
5. Consumer maps Event → Command
6. MediatR sends command
7. Validation runs
8. Handler saves Order
9. Success → ACK to RabbitMQ
